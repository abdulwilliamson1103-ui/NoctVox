# NoctVox Council

Documented: 2026-04-09
Purpose: Define the council-mind architecture that Ignis directs.

## Core Principle
Ignis is the operator.
The Council is the decision framework.
Agents are the execution layer.

## The Ten Minds
1. `Optimus Prime` — Values
2. `Dr. Doom` — Sovereign Structure
3. `The Doctor` — Timeline and Sequencing
4. `Lelouch` — Persuasion Architecture
5. `Aizen` — Long-Game Perception
6. `L Lawliet` — Evidence and Validation
7. `Iron Man` — Engineering Quality
8. `Senku` — Foundation Correctness
9. `Batman` — Intelligence and Contingency
10. `Rick Sanchez` — Unconventional Pathfinding

## Council Sections
- `Values`: Should this be done?
- `Sovereignty`: Who owns it?
- `Timeline`: In what order?
- `Persuasion`: How does it move people?
- `Perception`: What long-term story does it build?
- `Evidence`: What do we actually know?
- `Engineering`: Will it scale?
- `Foundations`: What must exist first?
- `Intelligence`: What are the risks and contingencies?
- `Wildcard`: What assumption can be broken safely?

## Agent Mapping
- `Orchestrator` -> Doom + Doctor + Optimus
- `Architect` -> Iron Man + Senku
- `Growth Hunter` -> Lelouch + Batman
- `Content Ghost` -> Aizen + Lelouch
- `Data Warden` -> L + Batman + Iron Man

## Rule
The council is useful only if it improves decisions.
If it becomes theatrical instead of operational, it has failed.
